Metodología en Espiral

Desarrollo del Sistema de Cuentas por Cobrar WEB

El programa de cuentas por cobrar fue concebido para centralizar la gestión de las deudas pendientes de los clientes, ofreciendo funciones clave como la actualización de deudas y la búsqueda eficiente de clientes.

Introducción

Desde el inicio se definieron objetivos claros: lograr una integración armoniosa del backend y el frontend, donde funciones lógicas fueran implementadas de manera confiable. Para alcanzar estos objetivos, se adoptó la metodología en espiral, la cual combina planificación, diseño, prototipado, pruebas y evaluación de riesgos en múltiples ciclos, permitiéndo adaptar y evolucionar con cada iteración.


Tecnologías clave utilizadas:
Node.js Express.js PostgreSQL Axios NodeCRON JavaScript HTML/CSS

Modelo de Desarrollo en Espiral

Ciclos de Desarrollo del Proyecto

Primera Iteración: Requisitos y Análisis Inicial

Objetivos:

  • Definir las necesidades del sistema: gestión de deudas, actualización de registros y búsqueda de clientes.
  • Definir los tableros en PostgreSQL relacionados a los requerimientos del sistema.

Análisis de Riesgos:

Tipado de datos erróneo Normalización incorrecta

Resultado:

Se completó el modelo de datos en la base de datos designada como "gestion_clientes".

Segunda Iteración: Diseño y Modelado del Backend

Objetivos:

  • Establecer la estructura inicial del proyecto determinando las tecnologías y herramientas a utilizar.
  • Desarrollo inicial del backend con: Node.js, Express.js, Axios, NodeCRON, etc.

Análisis de Riesgos:

Integración de datos inconsistente Acceso a métodos HTTP

Resultado:

Se completó un diagrama de flujo preliminar y se realizó una primera evaluación de la arquitectura del backend usando Postman.

Tercera Iteración: Diseño del Frontend y Prototipado

Objetivos:

  • Desarrollar un prototipo funcional tanto del backend como del frontend.
  • Implementar funciones claves en un entorno controlado para validar su comportamiento.

Análisis de Riesgos:

Validación de datos Compatibilidad en integración

Resultado:

Se creó un prototipo que permitió simular el comportamiento del sistema. Se identificaron áreas de mejora en respuestas del servidor y usabilidad.

Cuarta Iteración: Refinamiento y Optimización

Objetivos:

  • Mejorar la robustez y rendimiento de las funciones del backend.
  • Incorporar mejoras visuales y de interacción en el frontend.

Análisis de Riesgos:

Inconsistencias en actualización Seguridad de información

Resultado:

Se realizaron ajustes aumentando estabilidad y precisión. El frontend se refinó para ofrecer una experiencia más fluida y moderna.

Quinta Iteración: Integración y Pruebas Completas

Objetivos:

  • Integrar todas las funcionalidades desarrolladas.
  • Implementar pruebas automatizadas y manuales para evaluar integridad, rendimiento y seguridad.

Análisis de Riesgos:

Interacciones imprevistas Protección ante amenazas

Resultado:

Se completaron las pruebas de integración, solucionando pequeños inconvenientes. El sistema puede actualizar y buscar datos de forma segura y robusta.

Para el FuturoSexta Iteración: Despliegue y Retroalimentación Continua

Objetivos:

  • Desplegar el sistema en un entorno productivo.
  • Establecer mecanismos de monitoreo y retroalimentación.

Análisis de Riesgos:

Problemas de escalabilidad Adaptación a sugerencias

Resultado esperado:

Despliegue exitoso con sistemas de monitoreo en tiempo real. Retroalimentación de usuarios para futuras mejoras.
